最強雲端便利貼：GitHub Gist

🛠 不想開專案包時的最佳選擇

當你只想記錄一個不到 50 行的演算法，或是快速分享昨天找到的特殊 CSS 設定值，如果在 GitHub 建立一個全新的 Repo (專案庫) 會顯得非常殺雞用牛刀。這時就該請出 Gist。

Gist 的特色與優點

  1. 極簡介面：如同線上版的記事本，沒有資料夾結構，只有純粹的檔案與程式碼。
  2. 具備版本控制：即使是超小型的便利貼，Gist 也默默在背後幫你記錄每一次按下的「儲存」，支援隨時退回歷史版本。
  3. 嵌入網誌神技 (Embed)：Gist 提供一句特殊的 <script> 標籤。如果把那段碼貼到你的個人網站，網站上就會出現一個帶有高亮色彩的漂亮程式碼框，連外掛都不用裝。
