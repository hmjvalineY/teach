👩‍🏫 教師教學應用

使用 GitHub Classroom 指派與批改作業

教師在教導 Python 基礎時，透過 GitHub Classroom 建立一份包含基礎架構的作業。系統會自動為每位學生複製一份專屬的 Repository（儲存庫）。

📋 建立作業模板 → 🔗 發送邀請連結 → 📦 自動複製 Repo → ✅ Autograding 自動評分

🛠 核心功能：自動評分 (Autograding)

  1. 教師在作業 Repo 中設定測試腳本（如 pytest）
  2. 學生提交程式碼（Push 到 GitHub）
  3. GitHub Actions 會自動執行測試腳本
  4. 系統立即告訴學生程式碼是否正確
  5. 教師可以在後台一覽所有學生的評分狀態

💡 效果：大幅節省教師批改時間，學生也能獲得即時回饋。

👨‍🎓 學生學習應用

期末專案的版本控制與協作

學生分組進行「校訂選修：網頁設計」的期末專案，學會使用業界標準的 Git 工作流程。

🌿 建立 Branch 分工 → 💻 各自開發 → 📤 提交 Pull Request → 🔀 合併程式碼

📝 協作實例

  1. A 同學建立 feature/frontend 分支，負責前端版面設計
  2. B 同學建立 feature/backend 分支，負責後端邏輯
  3. 各自完成後提交 Pull Request 將進度合併到 main 分支
  4. 若程式碼發生衝突，透過 GitHub 介面解決
  5. 過程中留下完整的開發紀錄，展現團隊合作歷程

💡 學習重點：學會版本控制、分工合作、程式碼審查等業界標準工作流程。
