打造個人專屬的數位學習歷程網站

不依賴學校制式系統，利用 GitHub Pages 免費架設個人專屬的「學習歷程檔案網站」。

📝 建站步驟

  1. 在 GitHub 上尋找開源的 Jekyll 或 Hugo 履歷模板
  2. Fork（複製）模板到自己的帳號
  3. 修改設定檔，填入個人資訊
  4. 分類上傳高中三年的成果：
    • 📚 專題報告（PDF 或 Markdown）
    • 📷 社團活動照片
    • 💻 程式作品連結
    • 📜 志工證明與獲獎紀錄
  5. 啟用 GitHub Pages，網站即刻上線

💡 升學加分：在大學申請時附上個人網址，展現資訊能力，讓教授留下深刻且專業的印象。

🤝 師生協作

校刊社的編輯部流程 (Editorial Workflow)

告別檔名叫做「期末報告_最終版_真的最後一版_改.docx」的夢魘。

📰 校刊社專業流程

  1. 將每一期文章以 Markdown 格式存在 GitHub
  2. 記者完成初稿後，提交 Pull Request
  3. 編輯長利用 GitHub 介面逐行審查
  4. 在需要修改的句子旁直接留言建議
  5. 一切討論在線上完成，修改歷程清晰可見

💡 這正是專業出版社與媒體使用的版本控制流程！

👨‍🎓 學生應用

辯論比賽的攻防稿共筆

辯論隊利用 GitHub 管理講稿版本，確保全隊上場前手上的講稿都是最新且同步的。

🎤 辯論協作流程

  1. 將「一辯稿」、「二辯稿」等放在 Repository 中
  2. 針對敵方可能提出的論點開 Issue 進行腦力激盪
  3. 有更好的數據支援時，提案修改稿件（Pull Request）
  4. 所有修改經團隊審閱後合併
  5. 上場前確保大家手上都是最新且同步的版本
👩‍🏫 教師應用

管理學生的「專題式學習 (PBL)」進度

老師要求每組學生在 GitHub 上開一個 Project Board 規劃期末專題。

📊 評量方式

  1. 老師只要點進各組的頁面，即可看到每位同學的分工與進度
  2. 能在特定任務卡片下方留言給予指導
  3. 達到「過程導向」的評量方式
  4. 不僅看最終成果，更重視團隊合作與時間管理的學習歷程
👩‍🏫 教師應用

打造共創型「班級維基百科」

歷史老師開啟 Repository 的 Wiki 功能，讓全班學生共同編寫知識庫。

📖 班級 Wiki 範例：台灣近代史大辭典

  1. 開啟 Repository 的 Wiki 功能
  2. 每個學生負責撰寫幾個歷史名詞的條目
  3. Wiki 支援 Markdown，可輕鬆插入圖片與超連結
  4. 共同打造出一本屬於該班級的數位專書
🎯 推廣建議：以上所有應用案例完全不需要寫任何程式碼。建議從 GitHub 網頁版介面入手，搭配 GitHub Desktop 圖形化工具，並強調 Markdown 語法的簡單易學，讓師生在 10 分鐘內就能上手。
