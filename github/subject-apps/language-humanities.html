寫作的「草稿到定稿」過程，本質上就是一種「版本控制」

👩‍🏫 教師教學應用

利用「差異比對 (Diff)」進行寫作指導與同儕互評

英文老師讓學生用 Markdown 格式繳交作文。學生修改後提交第二版時，老師可使用 Diff 功能一目了然地看到修改內容。

📊 Diff 差異比對模擬

📝 essay_climate_change.md — 學生修改前後對比
Climate change is a big problem in the world today.
- Many people does not care about it.
+ Many people do not seem to care about this issue.
We need to take action before it is too late.
- The earth is getting more hotter every year.
+ The earth is getting increasingly warmer each year.

🛠 操作流程

  1. 學生使用 Markdown 格式撰寫英文作文並提交到 GitHub
  2. 老師給予初步修改建議（可直接在程式碼行留言）
  3. 學生根據建議修改後提交第二版（新的 Commit）
  4. 老師使用 Diff 功能查看修改處：
    • 🟢 綠色代表新增的文字
    • 🔴 紅色代表刪除的文字
  5. 也可透過 Pull Request 讓學生進行同儕互評 (Peer Review)
👨‍🎓 學生學習應用

共筆協作與開源翻譯

學生組成自主學習小組，將國外一篇關於「氣候變遷」的優質開源英文文章翻譯成中文。

📝 協作翻譯流程

  1. 在 GitHub 上建立翻譯專案 Repository
  2. 利用 Wiki 功能或共同編輯 Markdown 文件
  3. 每人負責不同的段落：
    • A 同學負責第 1-3 章
    • B 同學負責第 4-6 章
    • C 同學負責附錄與詞彙表
  4. 遇到翻譯瓶頸時，開 Issue 進行討論與集體決策
  5. 最終完成一份高品質的共筆翻譯作品

💡 學習價值：培養英語閱讀能力、團隊協作能力，以及翻譯的精確度與溝通技巧。
