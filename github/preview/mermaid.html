在 Markdown 繪製 Mermaid 圖表

🛠 實際操作與示範程式碼

繪製流程圖還要打開 Visio 或用繪圖軟體畫完再存成 PNG？在 GitHub 只要敲文字就好！

操作法：在你的 README.md 等 Markdown 檔案內，利用三個反引號標註 mermaid，然後以純文字撰寫邏輯。

📌 範例：畫一個簡單流程圖

```mermaid
graph TD;
    A[專案構思] --> B{是否使用 GitHub?};
    B -- 是 --> C[建立 Repo];
    B -- 否 --> D[改用一般資料夾];
    C --> E[享受美好的預覽功能];
```

📌 範例：開發甘特圖

```mermaid
gantt
    title 教學專案開發進度表
    dateFormat  YYYY-MM-DD
    section 設計階段
    系統架構設計     :a1, 2025-01-01, 30d
    section 開發階段
    前端切版實作     :after a1, 20d
```

GitHub 的文件解析器在展示這些語法時，會自動把它們渲染成漂亮的高解析度圖片！
