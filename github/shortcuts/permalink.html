凍結時間的網址（按下 y）

🛠 實際操作示範

傳送程式碼連結給別人時，最怕的就是「連結失效」或是「行號錯置」。

  1. 在 GitHub 點開一份程式碼檔案，目前網址列通常顯示 .../blob/main/檔案.js。
  2. 在該網頁按下鍵盤的 y 鍵。
  3. 觀察網址列，main 字眼瞬間被轉換成一長串 Commit Hash 隨機碼。
  4. 此時再複製網址並傳送給朋友。

💡 為什麼要這樣做？

main 是浮動的分支名稱。如果你傳送第 45 行的連結給別人，明天其他工程師修改了檔案新增了 10 行，別人點進去看到的第 45 行就完全不是你當初想分享的東西了！而按下 y 產生的 Permalink 會永遠死守在產生的那一秒鐘的版本。
