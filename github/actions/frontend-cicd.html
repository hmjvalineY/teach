前端 CI/CD 自動化部署

🛠 讓 GitHub 成為你的 DevOps 工程師

在過去，當你用 React、Vue 或 Angular 開發完網頁後，你需要手動在本地端執行 npm run build，然後把產生出來的 dist 或 build 資料夾手動上傳到伺服器。這個過程繁瑣又容易出錯。

現在，你可以透過 GitHub Actions 打造一條「無人值守」的全自動生產線（CI/CD）：

自動化流程大揭密：

  • 觸發條件： 當工程師把最新的程式碼推上 (Push) main 分支時。
  • 準備環境： GitHub 會在雲端開一台全新的伺服器，自動幫你安裝 Node.js。
  • 下載相依： 自動執行 npm install 下載所有套件。
  • 編譯打包： 自動執行 npm run build 將框架編譯成靜態網頁。
  • 自動部署： 將編譯好的結果自動發布至 GitHub Pages，一氣呵成！

從此之後，你只需要專心寫 Code 並 Push 到 GitHub，過幾分鐘後，全世界的使用者就能自動看到最新版本的網站了！

🎯 實作任務與示範：

我已經幫您在專案的 .github/workflows/ 資料夾內建立了一份名為 demo-react-cicd.yml 的腳本，裡面清楚記載了如何撰寫上述提到的每一個步驟。

  1. 透過 GitHub Desktop 將這個流程腳本連同您的專案上傳。
  2. 來到 GitHub 網站點擊正上方的 「Actions」 頁籤，您就能看到雲端伺服器正在依照腳本的指令，一步步編譯您的網站！
