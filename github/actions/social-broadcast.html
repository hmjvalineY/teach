社群全自動廣播網路

🛠 行銷傳播的好幫手

很多開源社群管理者每次發布了新版本的軟體，都要手動去 Discord, Twitter, 等各大平台發公告，這時候只要寫一隻 GitHub Action 就能一勞永逸。

觸發條件設定

這次我們不用 cron 時間排程，我們改用 release 作為觸發條件。當有人在 GitHub 按下發布新版本的按鈕時：

name: 廣播新版本
on:
  release:
    types: [published] # 當發布的時候執行

jobs:
  notify-discord:
    runs-on: ubuntu-latest
    steps:
      - name: 傳送訊息至 Discord 伺服器
        env:
          DISCORD_WEBHOOK: ${{ secrets.DISCORD_WEBHOOK }}
        uses: Ilshidur/action-discord@master
        with:
          args: "🎉 我們的專案釋出了新版本：{{ EVENT_PAYLOAD.release.name }}！快來看看吧！"
