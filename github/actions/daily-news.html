每日定時爬蟲與資料擷取

🛠 Actions Cron 排程實戰

GitHub Actions 就像免費的伺服器，只要你定義了排程設定檔，即使你的電腦處於關機狀態，GitHub 伺服器依舊會在指定時間為你執行任務。

YAML 範例檔：每天早上執行 Python

此腳本位於 .github/workflows/daily.yml：

name: 🤖 每日新聞自動剪報
on:
  schedule:
    # 此處時間為 UTC，如果你住在台灣，時間要加 8 小時
    # 這是指 UTC 的凌晨 4 點鐘執行 (即台灣中午 12 點)
    - cron: '0 4 * * *'

jobs:
  spider:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      
      - name: 啟動 Python
        uses: actions/setup-python@v4
        with:
          python-version: '3.9'
          
      - name: 開始抓取新聞
        run: |
          pip install requests
          python fetch_news.py
          
      - name: 將整理好的新聞 HTML 存回 GitHub
        run: |
          git config --global user.name "Actions Bot"
          git config --global user.email "bot@example.com"
          git add report-today.html
          git commit -m "update news"
          git push

Python 爬蟲腳本：fetch_news.py

這是一個簡單的 Python 腳本範例，會在背景擷取新聞並自動產生 HTML 檔案 (report-today.html)：

import requests
from datetime import datetime

# 取得開放資料或新聞 API (以維基百科每日精選文章為例)
today = datetime.today().strftime('%Y/%m/%d')
response = requests.get(f'https://zh.wikipedia.org/api/rest_v1/feed/featured/{today}')
data = response.json()

# 整理資訊並產生 HTML 內容
html_content = f"<html><head><meta charset='utf-8'><title>今日新聞剪報</title></head><body>\n"
html_content += f"<h1>🗞️ 今日新聞剪報 ({today})</h1>\n"

if 'tfa' in data: # 主要推薦條目
    title = data['tfa']['title']
    extract = data['tfa']['extract']
    url = data['tfa']['content_urls']['desktop']['page']
    
    html_content += f"<h2>{title}</h2>\n"
    html_content += f"<p>{extract}</p>\n"
    html_content += f"<a href='{url}' target='_blank'>閱讀完整文章</a>\n"

html_content += "</body></html>"

# 輸出儲存成檔案
with open('report-today.html', 'w', encoding='utf-8') as f:
    f.write(html_content)
    
print("新聞報表產生完成！")

透過這樣的方法，不需花費一毛錢，只需撰寫簡單的 Python 邏輯，即可完成無伺服器化的長期追蹤機器人，每天將最新的資訊推送到你的 Repo 裡！
