揪出戰犯神器 (Git Blame)

🛠 這行 Bug 究竟是誰寫的？

這是一個惡名昭彰但又非常實用的按鈕。當你在看過往專案時，心中浮起了「天啊，為什麼這一行 CSS 的寬度要設成這樣？」的疑惑時，這按鈕就能給你解答。

功能位置與效果：

  1. 在 GitHub 網站上點進你要檢視的檔案內文。
  2. 在右上角介面的排版選項旁邊，你會看到一個叫 Blame 的按鈕。
  3. 用力按下去。
  4. 畫面會瞬間一分為二：右邊保留程式碼，而左邊則會逐行標示出：「這一行程式碼最後一次修改，是誰在幾年幾個月前進行的，當時他寫了什麼標題」。

這功能除了可以用來揪出闖禍的同事（XD）之外，更重要的是它能帶你穿越時空，讓你透過作者留下的 Commit 標題，理解那一行怪異的設定當時是為了修復什麼更底層的問題而存在的。
