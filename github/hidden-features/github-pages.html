免費建立專屬的靜態網站：GitHub Pages

🌐 把程式碼變成活生生的網頁

GitHub Pages 讓你不需要另外購買虛擬主機，就能免費、快速地將 GitHub Repository 的內容轉化成一個靜態網站。非常適合用來架設個人作品集 (Portfolio)、技術部落格 (Blog) 或是專案的官方說明文件。

快速上手指南：

  • 開啟功能： 進入 Repository 首頁上方的 Settings > 左側選單找到 Pages。
  • 選擇來源： 在 Build and deployment 設定區域中，選擇 Deploy from a branch（通常選擇 main 或是 gh-pages 分支）。
  • 預設網址： 部署完成後，網站的預設網址格式會是 https://<你的帳號>.github.io/<專案名稱>。
  • 自訂網域： 如果你有自己購買的網域 (Custom domain)，也可以直接在這邊進行綁定，讓網址顯得更加專業！

進階玩法：

除了只寫原生的 HTML / CSS 之外，GitHub Pages 原生就支援 Jekyll 靜態網站產生器。當然，現在更流行的是結合 GitHub Actions，自動把 React、Vue 或 Angular 這些前端現代框架編譯出來的網頁部署上去，達成強大的自動化 CI/CD 無人值守流程！
